### HW3. Адресный план
``` 
Сделать адреcный план для окружения с использованием 4 подсетей из "24 битного блока" RFC 1918 и 2 сетей из RFC 6598.
Для сетей из RFC 1918 использовать подсети, третий октет которых начинается с 154.
Две сети должны иметь сетьевую маску 255.255.255.192, две другие сети — префикс /25.
Для сетей из RFC 6598 должно быть задействовано адресное пространство не менее, чем на тысячу IP адресов
```
``` 
### 128 64 32 16 8 4 2 1
Частное адресное пространство:
10.0.0.0/8 (10.255.255.255)
172.16.0.0/12 (172.31.255.255) 
192.168.0.0/16 (192.168.255.255)

Берем сеть из частного адресного пространства и делим ее на 4 сети (две с /26, две с /25)
172.16.154.0/26 (255.255.255.192)

/32-26 = 6 
2^6 = 64
64 - 2(адрес сети и broadcast адрес) = 62 хоста в сети

/32-25 = 7
2^7 = 128
128 - 2(адрес сети и broadcast адрес) = 126 хоста в сети

1) 172.16.154.0/26

Network:   172.16.154.0/26
HostMin:   172.16.154.1
HostMax:   172.16.154.62
Broadcast: 172.16.154.63

2) 172.16.154.64/26

Network:   172.16.154.64/26
HostMin:   172.16.154.65
HostMax:   172.16.154.126
Broadcast: 172.16.154.127

3) 172.16.154.128/25

Network:   172.16.154.128/25
HostMin:   172.16.154.129
HostMax:   172.16.154.254
Broadcast: 172.16.154.255

4) 172.16.155.0/25

Network:   172.16.155.128/25
HostMin:   172.16.155.1
HostMax:   172.16.155.126
Broadcast: 172.16.155.127

Данная подсеть рекомендована согласно RFC 6598 для использования в качестве адресов для CGN (Carrier-Grade NAT):
100.64.0.0 — 100.127.255.255 (маска подсети 255.192.0.0 или /10)

2^10 = 1024 
/32 -10 = 22

1) 100.64.0.0/22

Network:   100.64.0.0/22
```
